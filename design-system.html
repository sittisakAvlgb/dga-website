Getting Started

เริ่มต้นใช้งาน (สำหรับนักพัฒนา)

CSS Framework ทั้งหมดอยู่ในไฟล์เดียว /assets/design-system.css — ลิงก์แล้วใช้ตัวแปร (Design Token) และคลาสคอมโพเนนต์ได้ทันที ไม่ต้องตั้งค่าเพิ่ม

1 ลิงก์ Framework

เพิ่ม <link> ไฟล์ design-system.css ในส่วน <head>

2 ฟอนต์อัตโนมัติ

ไทยใช้ Kanit · อังกฤษใช้ Ubuntu ผ่าน --font-base (ใส่ lang="en" เพื่อสลับ)

3 ใช้คลาส / Token

ใช้คลาส เช่น .btn หรือเรียกตัวแปร var(--c-action) ใน CSS ของคุณ

ติดตั้ง + ตัวอย่างการใช้งาน 
<!-- 1) ลิงก์ Framework ในส่วน head -->
<link rel="stylesheet" href="/assets/design-system.css">

<!-- 2) ใช้คลาสคอมโพเนนต์ได้ทันที -->
<button class="btn btn--primary">ปุ่มหลัก</button>

/* 3) หรือเรียก Design Token ใน CSS ของคุณ */
.my-card {
  background: var(--c-surface);
  border: 1px solid var(--c-border);
  border-radius: var(--radius-lg);
  padding: var(--sp-5);
  box-shadow: var(--shadow-2);
}

Design Tokens

ตัวแปร CSS (Custom Properties) ของระบบ — เรียกใช้ด้วย var(--ชื่อ) ได้ทุกที่ และปรับธีมจากจุดเดียว

Tokenค่าการใช้งาน
--font-th'Kanit'ฟอนต์ไทย (ค่าเริ่มต้น)
--font-en'Ubuntu'ฟอนต์อังกฤษ (ใช้กับ lang="en")
--font-basevar(--font-th)ฟอนต์หลักของระบบ
--fs-display2.99remตัวเลข/ข้อความใหญ่ใน Hero
--fs-h1 / h2 / h32.49 / 1.73 / 1.44remลำดับหัวเรื่อง
--fs-body / small1 / 0.875remเนื้อความ / ข้อความเล็ก
--lh-base1.6line-height (รองรับวรรณยุกต์ไทย)
--sp-1 … --sp-84 · 8 · 12 · 16 · 24 · 32 · 48 · 64pxระยะห่างตามกริด 8px
--radius-sm/md/lg/xl8 / 12 / 18 / 24pxความโค้งมุม
--shadow-1/2/33 ระดับความลึก/การลอยตัวของการ์ด
--c-action #C2410Cพื้นปุ่มหลัก (ข้อความขาว 4.9:1)
--c-primary-50 #FFF3ECtint พื้นอ่อน (hover/แอคเซนต์)
--c-surface / bg / sunkenพื้นผิว 3 ระดับการ์ด / พื้นเพจ / พื้นพรีวิว
--c-focus #13497Aวงโฟกัส (WCAG 2.4.11)
--container1440pxความกว้างสูงสุดของเนื้อหา
เรียกใช้ Token ใน CSS 
.cta {
  font-family: var(--font-base);
  color: #fff;
  background: var(--c-action);
  padding: var(--sp-3) var(--sp-5);
  border-radius: var(--radius-md);
}

ฟอร์ม (Forms)

ทุก input มี label · error ผูก aria-describedby + aria-invalid (WCAG 3.3.1)

กรอกตามบัตรประชาชน

รูปแบบอีเมลไม่ถูกต้อง

โค้ดสำหรับใช้งาน 
<div class="field">
  <label for="name">ชื่อ-นามสกุล</label>
  <input class="input" id="name" type="text">
  <p class="field__hint">ข้อความช่วยเหลือ</p>
</div>

<!-- สถานะ error (ผูก aria เพื่อ WCAG 3.3.1) -->
<div class="field">
  <label for="email">อีเมล</label>
  <input class="input" id="email" type="email"
    aria-invalid="true" aria-describedby="email-err">
  <p class="field__error" id="email-err">รูปแบบอีเมลไม่ถูกต้อง</p>
</div>

Standard · Accessibility

Accessibility (WCAG 2.2 AA)

ออกแบบให้ผ่าน AA ตั้งแต่ระดับระบบ ทั้งโหมดสว่างและมืด — ตรวจด้วย axe / Lighthouse / WAVE / NVDA

  • Contrast ข้อความ ≥ 4.5:1 ทั้ง 2 ธีม
  • โฟกัสมองเห็นชัดด้วย :focus-visible (2.4.11)
  • เข้าถึงทุกอย่างด้วยคีย์บอร์ด + skip link (2.1.1, 2.4.1)
  • แท็บ/Modal ทำงานตามแพตเทิร์น ARIA · Target ≥ 44px (2.5.8)
  • เคารพ prefers-reduced-motion (2.3.3)
